Visualizing Travel Behavior Data Using Python and flowmap.blue


I developed a script that intakes Replica origin/destination data, processes it, and creates an output that can be plugged directly into flowmap.blue to visualize the origin/destination travel flows of over nine million average weekday trips within Alameda County.

Replica origin/destination data is created through statistical modeling of anonymized mass mobile location data and U.S. Census data, and provides accurate flow counts at the census tract level for the entire U.S. The Python script I created can therefore be used to efficiently create a similar interactive map for any area or city of the U.S.

Included below is the resultant flowmap.blue map, followed by the actual Python code used to parse the Replica data and create the flowmap.blue compatible output. The code is well documented: it contains a summary of steps taken, a to-do list to aid in collaboration and guide future efforts, a list of outstanding questions, and a brief description for each section of code.
